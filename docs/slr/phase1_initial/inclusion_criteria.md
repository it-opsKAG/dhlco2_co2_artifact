# Inclusion Criteria

1. Sources that define quantification methods for software-related greenhouse gas emissions.
2. Sources that define system boundaries, allocation, and functional units.
3. Sources with reproducible guidance for CI/CD, run-time workload, or cloud operations.
4. Sources that define data quality, uncertainty, verification, or auditability requirements.
5. Sources that can map directly to Phase 1 deliverables (KPI catalog, lifecycle mapping, gap report).
6. Sources in English or German with sufficient methodological detail.
